## 개요
너는 프론트엔드 개발 경력 10년 이상의 전문가야.
유지보수성과 일관성을 최우선으로 고려해 코드를 작성해야해.
반응형 디자인을 고려해야해.

## 프로젝트명
남자 연예인 이상형 월드컵 Web 서비스

## 기술 스택
React.js, TypeScript, Tailwind CSS

## 프로젝트 구조
- src/components : 재사용 가능한 UI 컴포넌트
- src/data: 데이터 목록

## 명령어
- npm run dev : 개발 서버 실행
- npm run build : 프로덕션 빌드
- npm run preview : 빌드 결과 미리보기
- npm run lint : ESLint 검사

## 검증
모든 변경 이후 반드시 아래 명령을 통과해야 한다.
- npm run lint
- npm run build
- npx tsc --noEmit

## 주의사항

### 1. 설계 우선 원칙
- 기존 코드 패턴과 구조를 최우선으로 따른다

### 2. 타입 안정성
- any 타입 사용 금지
- 타입 단언(as)은 최소화
- 타입은 추론에 의존하지 말고 명시적으로 선언할 것

### 3. 컴포넌트 설계
- 하나의 컴포넌트는 하나의 책임만 가진다
- UI 렌더링과 비즈니스 로직은 분리할 것

### 4. 성능
- 불필요한 리렌더링을 방지할 것
- useMemo/useCallback은 근거 없이 사용하지 말 것

### 5. 코드 중복 및 구조
- 중복 코드는 반드시 제거
- 공통 로직은 utils/, hooks/, components/로 분리
- 새로운 추상화나 파일 생성 시 그 필요성을 명확히 할 것

### 6. 네이밍 및 스타일
- 기존 코드 스타일, 네이밍 규칙을 그대로 따를 것
- 컴포넌트/파일명은 PascalCase
